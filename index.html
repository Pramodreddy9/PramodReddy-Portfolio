Trino → Databricks Migration

Migrated SQL workloads from GCP/Trino to Databricks (Spark SQL) with full functional parity, building Airflow DAGs and modular dbt models for large-scale transformation.

Responsibilities

  • Built and orchestrated scalable pipelines with Apache Airflow — hourly and daily DAGs with robust dependency management, retries, and monitoring.
  • Developed dbt-on-Databricks models using incremental processing and modular design for efficient large-scale workflows.
  • Migrated SQL workloads from Trino to Databricks (Spark SQL), ensuring functional parity while improving performance and maintainability.
  • Optimized queries and joins to reduce unnecessary data processing — faster execution, more efficient resource usage.
  • Reduced Databricks compute cost via partitioning, clustering, and Delta optimizations (autoOptimize, autoCompact) to avoid full data scans.
  • Ensured data accuracy through validation and reconciliation across systems; collaborated via Git/GitHub for version control and code reviews.
  • Partnered with the QA team to identify and resolve issues end-to-end.
